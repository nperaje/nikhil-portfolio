② Transform — SQL Techniques
⚙️
Advanced SQL Layer
Transformation & Data Engineering
Multi-layered SQL logic handles JSON parsing, currency conversion, billing-cycle normalisation, data quality filtering, and NULL-safe arithmetic across all billing models.
Technique Function / Pattern Purpose
JSON Parsing JSON_EXTRACT_SCALAR, JSON_VALUE Extracts pricing fields stored as semi-structured JSON in pricing_models.details
CTE Layer WITH raw AS (...) Separates raw extraction and joins from the presentation layer for readability and reuse
Multi-model CASE CASE WHEN model_type = ... Branches pricing logic across four billing models: per_seat, fixed, metered, one_time
Currency Normalisation (price * quote_rate) / base_rate Converts all contract currencies to USD for apples-to-apples benchmarking
Billing Cycle Normalisation SAFE_DIVIDE, DATE_DIFF, REGEXP_EXTRACT Converts monthly, quarterly, annual, and one-time frequencies into a unified monthly cost
String Manipulation REGEXP_REPLACE, TO_JSON_STRING, SUBSTRING Cleans JSON-encoded numeric strings and safely casts values for arithmetic
License Bucketing SAFE_CAST + BETWEEN in CASE Groups license counts into 14 ordered tiers (1–10, 11–20 … 1M+) for histogram benchmarking
Aggregate & Dedup SUM, COUNT, STRING_AGG(DISTINCT ...) Rolls up contract credits and file names per service period at pricing model grain
Data Quality Filter REGEXP_CONTAINS, NOT LIKE ANY Excludes sandbox, demo, test workspaces and personal email domains from benchmark
NULL Safety COALESCE, SAFE_DIVIDE, SAFE_CAST Handles NULL pricing values, unlimited-seat flags, and division-by-zero across all branches
Date Arithmetic DATE_DIFF, EXTRACT(YEAR ...), DATE() Computes contract length in days, term buckets in months, and service start year
